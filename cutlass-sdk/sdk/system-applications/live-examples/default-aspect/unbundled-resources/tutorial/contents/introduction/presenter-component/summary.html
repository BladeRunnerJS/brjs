To use Presenter, you need to create two things: the HTML template with data-bind attributes and the Presentation Model. You then create a Presenter Component using them. You can read more about the Presenter Component in the Creating a Presenter component, and a detailed example showing you how to do it is provided in the Presenter Worked Example.

The HTML Template (View)

The View is just a fragment of HTML5, with data-bind attributes to connect DOM elements to JavaScript objects in the Presentation Model. The file containing the View, along with other HTML resources, is stored in a blade's resources/html directory, and the top-level HTML element within the View must have a correctly formatted id attribute, in order to be properly referenced by the Presenter Component.

See Creating a Presenter component for further details.

Presentation Model

The Presentation Model JavaScript file sits in a folder with the same name as the blade it belongs to, under the /src/ directory.
E.g: test-bladeset/blades/myblade/src/ ... /myblade/pres_model.js

The file has to be set up as a presentation model by including a class that extends caplin.presenter.PresentationModel.

See Creating a Presenter component for further details.

Binding them together into a Presenter Component

To do this, you need to create an instance of a caplin.presenter.component.PresenterComponent that uses the HTML template and the presentation model defined above. There are three ways of including the Presenter Component in an application or webpage:

  1. Create an instance in JavaScript directly
  2. Use the Caplin component factory
  3. Define it using Webcentric configuration
