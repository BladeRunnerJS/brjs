The caplin.presenter.node.SelectionField class is a control model that allows the user to choose a single item from a list of options.

Internally, options are stored within a caplin.presenter.node.OptionsNodeList object, which extends NodeList. This allows you to use sub-templates to render each option, while the View is automatically refreshed when the options are updated.

When pairing with a <select> element, the options binding must appear before the value binding, otherwise Knockout will invalidate the model on first use, potentially causing problems for other code that relies on it.

This example shows how a single SelectionField can be rendered, using both <select> and <input type="radio"/> elements at the same time. You can specify which option is initially selected by adding a second argument to the SelectionField.

For example: you can make "Films" the selected option by modifying the hobbies constructor, so that it reads: caplin.presenter.node.SelectionField(pHobbies, "Films");
