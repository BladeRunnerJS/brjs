The Control Plugin allows custom JavaScript controls to be written and bound to a View template. For information about the control implementations provided by Caplin see the Custom Controls section.

To add a custom control to the View template, use the control binding attribute, followed by the logical alias name of the custom control, as shown below: 

<div data-bind="control:'my-control',controlNode:buysell">

Note that the value binding for the control plugin has been deprecated, and the controlNode binding will now be used instead. The html element where the control plugin is defined is now reused by the control plugin avoiding nesting more elements. Therefore it is not recommended to use the value binding, as it is a reserved knockout binding that can clash with other presentation node's bindings.

To create a custom control write a class that implements the caplin.presenter.control.ControlAdaptor interface, and then create a aliasDefinitions.xml in the same bladeset's resources directory. This will allow BladeRunner to automatically register your new control: 

<aliasDefinitions xmlns="http://schema.caplin.com/CaplinTrader/aliasDefinitions">
    <alias name="my-control" defaultClass="novobank.MyControl" />
</aliasDefinitions>

To learn more about aliasing read the Aliasing documentation.

The custom control is passed an instance of PresentationNode, which is the "model" of the control. The control is responsible for creating an HTML DOM element, and rendering itself appropriately. It creates whatever 2-way bindings are required. These bindings will...

  • Add DOM event listeners, to receive user gestures and update its model.
  • Add listeners to its model and update the HTML DOM elements.
