It's sometimes useful to be able to restore a user interface to a previous state, such as when you want a trade ticket to be reset to its initial state when a trade completes. This can be achieved by creating a collection of all the properties that make up the Presentation Model, and using the caplin.presenter.property.Snapshot class, to store the current values of each of them. Later the snapshot can be restored, using the apply() method.

Only writable (or editable) properties are stored within snapshots, since simple properties can't be changed from their original values in any case.

In this example, the initial state of the ticket is saved at the end of the constructor. Note how the properties() method is invoked with no arguments, causing all descendent properties to be stored.
