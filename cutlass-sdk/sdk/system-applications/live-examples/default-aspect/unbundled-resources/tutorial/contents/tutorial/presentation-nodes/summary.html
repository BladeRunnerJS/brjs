A presentation node is a class that contains a set of properties and/or other presentation nodes. It extends the caplin.presenter.node.PresentationNode class, and behaves as a "mini" presentation model, for a small part of the View. A presentation node can be a single, pre-defined object (such as a Field), or it can resemble the main Presentation Model, containing several additional nodes and properties.

Presentation nodes let you manipulate several related properties at once; reducing the amount of code you have to write. This provides a number of advantages.

  • Grouping logically related properties makes your code easier to understand.
  • Many properties are created in a single node - reducing amount of code you need.
  • It helps standardise the structure and naming of properties - making utilities more useful.

There are quite a lot of predefined nodes within Presenter, including caplin.presenter.node.Field, which has already been mentioned. What might not be immediately obvious is that caplin.presenter.PresentationModel is itself a node, and like other nodes it is an extension of the caplin.presenter.nodes.PresentationNode class.

  • caplin.presenter.PresentationModel
  • caplin.presenter.node.Field

Instances of Presentation Node classes are created in a tree structure, with the Presentation Model at the tree's root, and the properties of each node as its leaves.
