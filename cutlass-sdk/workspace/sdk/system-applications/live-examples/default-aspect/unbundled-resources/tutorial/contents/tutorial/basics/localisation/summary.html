Presenter's localisation facilities let you configure your app to be displayed in multiple languages, with the correct content, numeric format and date format for the selected locale (which is usually determined by the locality settings on the user's browser). Each supported locale must have its own .properties file containing the translated versions of each localisation key.

Presenter lets you localise your app in three ways:

HTML Template Localisation Tokens:
You place these tokens within the HTML View template, inside curly brackets and prefixed by @, and they are replaced by the localised value for that token.
For example: @{tutorial.amount.instruction}
API Translation
The Presentation Model calls the ct.i18n() method of the Caplin Localisation Library, using the result to set the value of a property. Note: ct.i18n() is a global function that provides a shortcut to the caplin.i18n.translator.getMessage() function.
Validators That Support Localisation
Some validators accept a localisation token in their constructor, which the Presentation Model can set to ensure a localised version of the error message is displayed when an error occurs.
