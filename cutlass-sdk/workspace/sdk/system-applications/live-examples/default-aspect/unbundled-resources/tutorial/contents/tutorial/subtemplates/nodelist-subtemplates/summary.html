Sometimes you'll want to render the same template for each item in a collection (e.g. rows in table or legs in a trade). You do this by specifying a caplin.presenter.node.NodeList within the foreach parameter of the template binding. Any changes made to the NodeList are automatically reflected on screen, with DOM nodes being created and destroyed as required.

This example creates a caplin.presenter.node.NodeList named legs, and populates it with two presentation nodes; one for each leg. You could of course create a named sub-template for this, as we did in the tutorial on Scoped Sub-Templates, and use it to iterate through all the items in the NodeList.

However, you don't actually need to define a new named sub-template, if the HTML fragment is not big enough to justify it. In this case, the two <span> tags that comprise the sub-template just sit inside the <div> that contains the template binding, and it iterates through the NodeList perfectly well.
