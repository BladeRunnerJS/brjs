When you bind a sub-template to an inner PresentationNode, you don't have to state the property bindings in full, each time; you can just refer to the properties within the template, by identifying which higher-level template owns them. This makes it easy to re-use the same sub-template for multiple instances of the same object, or for objects with the same properties.

In earlier versions of Presenter, if you wanted a sub-template to use properties from a higher level template, you had to declare each required property explicitly in the template{} binding, within a set of "templateOptions". Using the scoping tags below is easier and neater all round.

In this example, the Presentation Model contains an account field, as well as two child presentation nodes (leg1 and leg2, which are instances of TradeLegPresentationNode, defined further down). The child nodes represent legs of a trade with different amount fields. A sub-template is then used to render the contents of each child presentation node, with a different scope each time.

You could also use the sub-template to refer to any of its ancestors' scopes using:

  • $parent or $parents[0] to refer to the presentation node associated with the current template's parent
  • $parents[1] to refer to the presentation node associated with the current template's grandparent, etc...
  • $root to refer to the presentation node associated with the topmost template
