"Manual input" is of course a euphemism for "potential error", so the Editable Properties class comes complete with a number of validators, to allow you to check that what a user has just entered is valid input.

Validators are classes that implement the caplin.core.Validator interface, and they can be added to an EditableProperty, to see if the input is valid. In this example, a NumericValidator has been added to the amount property, to make sure that no non-numeric characters are entered. The content of the error message ( tutorial.invalid.number) is stored in a .properties file, see the tutorial on Localisation for further details.

The value of the property changes, even if the validation fails. This is because a Presentation Model is a one-to-one mapping with the screen. Be careful not to propagate the error value into the Domain Model.
